# Showing Your Project to Your Adviser (Option 2: ngrok)

This guide will help you set up **ngrok** to create a temporary public URL for your local project, allowing your adviser to access it from anywhere.

## Prerequisites
- Your local web server (e.g., XAMPP, Apache) must be running.
- You can access your project at `http://localhost/CommunaLink` (or your local URL).

## Step 1: Download and Setup ngrok
1. Go to [ngrok.com](https://ngrok.com/) and sign up for a **free account**.
2. Download the ngrok agent for Windows.
3. Extract the `ngrok.exe` file (usually to your `Downloads` or `Desktop`).
4. Open a terminal (Command Prompt or PowerShell) and navigate to the folder where you extracted ngrok.
5. Authenticate your agent (find your authtoken in the ngrok dashboard):
   ```bash
   ./ngrok config add-authtoken YOUR_AUTHTOKEN
   ```

## Step 2: Start the Tunnel
1. Run the following command to start a tunnel to your local Apache server (usually on port 80):
   ```bash
   ./ngrok http 80
   ```
2. You will see a terminal screen with a "Forwarding" URL (e.g., `https://a1b2-c3d4.ngrok-free.app`). **Copy this URL.**

## Step 3: Update Your .env File
To ensure all links and assets in the application work correctly, you must update the `APP_URL` in your `.env` file.

1. Open your `.env` file in the root directory.
2. Add or update the `APP_URL` line:
   ```env
   APP_URL=https://a1b2-c3d4.ngrok-free.app/CommunaLink
   ```
   *(Replace the URL with the one generated by ngrok. Include the `/CommunaLink` subfolder if that is how you access it locally.)*

## Step 4: Share the Link
1. Copy the full URL (including the subfolder): `https://a1b2-c3d4.ngrok-free.app/CommunaLink`
2. Send this link to your adviser!

> [!IMPORTANT]
> The tunnel will only work as long as your terminal/command prompt window remains open and your computer stays awake. If you close it, the URL will expire and you will get a new one next time you run the command.

## Troubleshooting
- **404 Not Found**: If the adviser sees a 404, double-check that the `APP_URL` in your `.env` matches the ngrok URL and that the folder name (e.g., `/CommunaLink`) is correct.
- **Connection Refused**: Ensure XAMPP/Apache is running on your machine.
